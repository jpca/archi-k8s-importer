# Contributing to K8S Importer for Archi

Community contributions like documentation, tests, fixes, new features and improvements are welcome.

Each contribution is expected in as a draft pull request (PR)
with a descritpion that references an opened issue with at least a positive reply from a code maintainer.

Each contribution must have :
- a branch with the naming convention "feature/<feature_name>_<issue_number>" or "fix/<fix name>_<issue_number>"
- a minimum of commits with a meaningfull message (use git squash on the branch before the PR)
- an automated test to check the contribution

AI is allowed if the human author masters the PR content

A code maintainer can refuse any contribution, with justifiable response to the contributor. 

By contributing, you agree that your contributions will be licensed under
the same Apache 2.0 License.
